# FalconNet Development Rules

## Stack
- Next.js 16
- TypeScript
- TailwindCSS
- Spring Boot
- MariaDB
- PWA

## Core Principles
- Mobile-first always
- Professional UI
- Facebook/Threads/X inspiration
- Clean architecture
- Reusable components
- Strict TypeScript
- No any
- Responsive on all devices
- Dark mode compatible
- PWA optimized

## Frontend Rules
- Avoid giant components
- Prefer composition
- Use reusable UI primitives
- Smooth animations only
- Keep layouts lightweight
- Optimize rendering
- Prevent hydration errors

## UX Rules
- Skeleton loaders
- Empty states
- Loading states
- Error states
- Smooth transitions
- Good tactile feedback

## Performance
- Avoid unnecessary rerenders
- Lazy load heavy components
- Optimize images/videos
- Minimize client components
- Keep bundle size optimized

## Validation
Before finishing:
- Run TypeScript checks
- Run ESLint
- Verify responsive
- Verify dark mode
- Verify mobile UI
- Verify no hydration errors

## Design Direction
FalconNet should feel:
- Modern
- Clean
- Elegant
- Fast
- Social
- Professional
