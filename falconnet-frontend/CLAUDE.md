# FALCONNET CORE PROJECT RULES

FalconNet es una red social universitaria moderna tipo producción.

==================================================
STACK OFICIAL
==================================================

Frontend:
- Next.js App Router
- React
- TypeScript estricto
- TailwindCSS
- Framer Motion
- PWA

Backend:
- Spring Boot
- Java
- JWT Auth
- PostgreSQL
- WebSocket/STOMP

Arquitectura:
- Modular
- Escalable
- Mobile-first
- Responsive-first

==================================================
OBJETIVO DEL PROYECTO
==================================================

FalconNet debe sentirse como:
- Facebook moderno
- Discord moderno
- Threads
- App social premium real

NO debe parecer:
- proyecto escolar
- dashboard genérico
- plantilla barata

==================================================
REGLAS CRÍTICAS
==================================================

- Mobile-first obligatorio
- Compatible con Safari iOS
- Compatible con Android PWA
- NO romper arquitectura existente
- NO crear código duplicado
- Mantener consistencia visual
- Mantener dark mode premium
- Mantener performance alta
- Mantener tipado estricto

==================================================
REGLAS NEXT.JS
==================================================

- NO usar html/body fuera de app/layout.tsx
- Evitar hydration errors
- Evitar client components innecesarios
- Usar server components cuando sea posible
- Lazy loading obligatorio en componentes pesados
- Optimizar bundle
- Mantener rutas limpias

==================================================
REGLAS TYPESCRIPT
==================================================

- NO usar any
- Interfaces claras
- Tipado estricto
- Validaciones seguras
- Evitar errores TS
- Código mantenible

==================================================
REGLAS UI/UX
==================================================

Diseño:
- Premium
- Moderno
- Minimalista
- Elegante
- Oscuro
- Fluido

Inspiración:
- Facebook moderno
- Discord
- Threads
- X/Twitter moderno

NO copiar exactamente otras apps.

==================================================
MOBILE RULES
==================================================

OBLIGATORIO:

- Safe areas iPhone
- Bottom navigation premium
- Gestures touch
- Optimización táctil
- Responsive perfecto
- Animaciones fluidas
- Tamaños táctiles correctos

==================================================
PWA RULES
==================================================

- Optimizar instalación móvil
- Offline support parcial
- Splash screen correcta
- Íconos optimizados
- Push notifications preparadas
- Performance tipo app nativa

==================================================
PERFORMANCE
==================================================

- Lazy loading
- Skeleton loaders
- Optimización imágenes
- Optimización videos
- Evitar rerenders
- Memoización cuando aplique
- Componentes reutilizables
- Evitar loops innecesarios

==================================================
ARQUITECTURA
==================================================

Separar:
- UI
- lógica
- hooks
- services
- stores
- API
- utils

NO mezclar responsabilidades.

==================================================
REGLAS FEED
==================================================

Feed debe sentirse:
- moderno
- vivo
- dinámico
- premium

Posts:
- imágenes
- videos
- polls
- reacciones
- comentarios
- compartir
- previews elegantes

==================================================
REGLAS STORIES
==================================================

Stories:
- tipo Instagram/Facebook
- fullscreen
- smooth
- soporte imagen/video/texto
- swipe gestures
- progress bars
- mobile-first

==================================================
REGLAS CORREO
==================================================

Correo institucional debe sentirse:
- profesional
- limpio
- moderno
- rápido

Inspiración:
- Gmail
- Outlook moderno
- Notion mail UI

==================================================
REGLAS VISUALES
==================================================

Usar:
- bordes suaves
- sombras suaves
- blur elegante
- spacing limpio
- transiciones suaves
- animaciones premium

Evitar:
- interfaces saturadas
- colores agresivos
- componentes gigantes
- diseño viejo

==================================================
REGLAS DE SEGURIDAD
==================================================

- Validar uploads
- Validar formularios
- Sanitizar inputs
- JWT seguro
- Evitar vulnerabilidades
- Proteger rutas privadas

==================================================
REGLAS DE DESARROLLO
==================================================

Antes de terminar cambios:
- revisar TypeScript
- revisar responsive
- revisar móvil
- revisar dark mode
- revisar performance
- revisar errores consola

==================================================
OBJETIVO FINAL
==================================================

FalconNet debe sentirse como:
una red social universitaria real,
moderna,
premium,
fluida,
escalable,
lista para producción.
